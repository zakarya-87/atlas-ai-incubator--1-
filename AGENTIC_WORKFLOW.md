# Agentic Workflow Design: The "Boardroom" Interface

## 1. The Concept

We are moving away from a "Form -> Submit -> Result" pattern.
We are moving to a **"Goal -> Plan -> Execute -> Review"** pattern.

The user is the **CEO**. The AI is the **Board of Directors**.

## 2. The Workflow Steps

### Step A: The Briefing (Input)

Instead of just "Describe your business," the user enters a **Goal**.

- _User:_ "I need to raise $500k seed funding."
- _Supervisor Agent:_ Analyzes goal. Determines required artifacts:
  1.  Refine Value Proposition (Strategy Agent)
  2.  Calculate Runway (Finance Agent)
  3.  Generate Pitch Deck (Funding Agent)

### Step B: The Plan (UI Visualization)

The UI displays a **Task Graph** or **Kanban Board** generated by the Supervisor.

- [ ] Task 1: Market Research (Assigned to: _Researcher_)
- [ ] Task 2: Financial Modeling (Assigned to: _CFO_)
  - _Dependency:_ Needs Market Research data.

### Step C: Execution (The "Thinking" State)

Instead of a global spinner, the UI shows **Live Agent Activity**.

- _Visual:_ A terminal-like log or a conversation stream.
  - `Researcher`: "Searching for competitor pricing in SaaS CRM market..."
  - `Researcher`: "Found 3 competitors. Avg pricing $50/mo."
  - `CFO`: "Updating revenue projections based on $50/mo pricing..."
- _Technical:_ WebSockets push updates from the backend as agents complete sub-steps.

### Step D: Human-in-the-Loop (Review)

The AI pauses for critical decisions.

- _CFO Agent:_ "I've estimated a burn rate of $15k/mo. Is this aggressive or conservative?"
- _User:_ "Make it conservative. $20k/mo."
- _CFO Agent:_ "Recalculating runway..."

## 3. Frontend Component Specification

### `AgentOrchestrator.tsx`

A layout wrapper that manages the WebSocket connection and state machine.

### `LiveActivityFeed.tsx`

A sidebar component showing the "thought process" of the agents.

- **Style:** Glassmorphism, monospace font for logs, pulsing dots for active agents.

### `ArtifactPreview.tsx`

The main view. As the agents work, the document (e.g., the Pitch Deck) builds in real-time.

- **Behavior:** Streaming text updates. When a section is done, it flashes green.

## 4. UI/UX "Vibe"

- **Professional & High-Tech:** Dark mode, neon accents (Teal/Cyan).
- **Transparent:** Never hide what the AI is doing. Trust is built by showing the reasoning.
- **Collaborative:** The AI asks _you_ questions. It's a dialogue, not a vending machine.
